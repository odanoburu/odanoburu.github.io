Developing grammars with GF

This is a short note to record a few pointers that should help a newcomer learn GF. In case you don’t know what GF is:

Grammatical Framework (GF) is a domain-specific programming language for grammar writing. It is a functional programming language, with syntax inspired by the Haskell programming language; it draws from intuitionistic type theory for its type system.

I wrote a short introduction to GF in my MSc thesis (which is about implementing a GF grammar for Portuguese), but the best source for learning GF is pairing the GF book with a read of Inari Listenmaa’s blog. If you don’t have so much time as to read a book, you can checkout the somewhat old GF tutorial or the GF Resource Grammar Library (RGL) tutorial (I’d pick the former if you are more of a programmer than a linguist, and the latter if the contrary).

Installing GF from binary is straightforward, and is probably the best course of action if you are not familiar with Haskell (the language of implementation) tooling. That’s all you need for playing around, but as soon as you want to contribute grammars you probably want to clone the RGL repository and build the latest versions of its grammars (you don’t need Haskell for that, only bash or cmd).

If you need help or have questions the best forum is probably the the gf-dev mailing list, but the #gf channel on freenode is good help too.
